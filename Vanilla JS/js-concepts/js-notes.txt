1. Global execution context is created when JS file starts running. 
   It has 2 parts - memory and thread of execution
2. Memory creation phase 
   - memory is allocated to variables and functions   
   - variables are set to undefined 
   - functions declarations are stored as full function definitions
3. Execution phase - assign values to variables. 
   If a function is there, create a new Execution context for the function. 
   Push the Function exec context (FEC) onto the call stack
   Then have memory creation and execution phases for that function.
   Function execution phase - executes the func body.
   Context is popped out of the stack.

let a = 1;
let b = 2;
function add(x, y) {
    let total = x + y;
    return total;
}
let result1 = add(a, b);
let result2 = add(10, 5);

Call stack - Code is stored line by line (first GEC, then let a and so on)

BigInt 
Number range is -(2^53 - 1) to 2^53 - 1. 
If we need to assign a value beyond this range, then we should opt for BigInt.

let b = 2n ** 54n; // BigInt (exact)

Generators v/s Async await:
- Return Type: Generators return an Iterable/Iterator object (which you can loop over); async functions always return a Promise.
- Control Flow: Generators provide manual execution—the function "pauses" at yield and only resumes when the caller triggers .next(). 
  async/await provides automatic execution—the engine manages the pausing/resuming logic once the awaited Promise settles.
- Primary Purpose: Generators are designed for custom iteration and lazy evaluation (generating infinite sequences or memory-efficient data processing);   
  async/await is specialized for linearizing asynchronous logic (making API calls read like synchronous code).

Prototypal Inheritance 
 - The Prototype Chain: Every object has an internal link, denoted as [[Prototype]]. When you access a property, the engine performs a recursive lookup: it checks the object, then its prototype, then the prototype's prototype, until it reaches Object.prototype and finally null.

 - Memory Efficiency: Instead of every instance of a 'User' having its own copy of a save() method, they all point to a single save function on the User.prototype.

 .prototype: This is a property of a Constructor Function. It is the "blueprint" or the object that will become the [[Prototype]] of all instances created with the new keyword.

__proto__ (or Object.getPrototypeOf): This is the actual link on the instance. It points to the prototype object it inherited from.

Defining a method in a class is identical to attaching it to the function's .prototype property.

Object.create(null) -> doesn't have any prototype

Hoisting:
var is hoisted (declared at the top of the scope) and initialized to undefined 
let/const are also hoisted but not initialized.
That's why we get ReferenceError cannot use variable before initialization

Temporal Dead Zone - start of the block where the variable is hoisted until the line where the variable is declared. Accessing the variable during this time gives ReferenceError. 

console.log('Hoisting let')
{
   // let a is already declared here due to hoisting, but not initialized yet 
   // So error is about initialization, not declaration
   console.log('Hoisting 1...');
   console.log(a); // ReferenceError cannot access a before initialization
   console.log('Hoisting 2...');
   let a;
}

let a // declaration 
a = 2 // initialization 

let a = 2 // declaration + initialization together
